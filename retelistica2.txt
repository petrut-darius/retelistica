Network Protocol Overview:
	Network Communications Protocols : transportul de date intre doua hostu-uri (TCP, HTTP, etc)
	Network Security Protocols : securizeaza data, authentication, data integrity si data encryption (SSH, SSL, TLS)
	Routing Protocols : aleg cea mai buna ruta dintre hosturi(OSPF, BGP)
	Service Discovery Protocols : detecteaza automat dispozitive sau servici (DHCP, DNS)
	
Network Protocol Functions:
	addressing : identifica senderu si receiveru mesajului (Ethernet, IPv4 si IPv6)
	reliability : mesajul va fi trimis sigur, indiferent daca este pierdut sau stricat in transmisia dintre hosturi (TCP)
	flow control : transferul de date se faca la o rata eficienta intre cele doua host-uri (TCP)
	sequencing : se da un index la segmente, astfel incat sa poata fi trimis si asamblat in ordinea corecta (TCP), segmentele vor fi retransmise daca nu ajung hostul destinatie
	error detection : verifica daca data a fost corrupted in timpul transmisiei (Ethernet, IPv4, IPv6 si TCP)
	application interface : informatii intre aplicatii network (layeru 7) (HTTP, HTTPS)

Protocol Interaction:
	HTTP : acest protocol guverneaza cum interactioneaza un web server si un client web
	TCP : garanteaza ca datele vor ajunge cum trebuie si ca se face la o rata eficienta
	IP : responsabil sa trimita mesajul intre sender si receiver, este folosit de routere ca sa transmita mesajul mai departe catre alte networkuri
	Ethernet : responsabil sa transmita data intre NIC si NIC intr-un LAN

Application Layer:

Name System

	DNS - Domain Name System. Translates domain names such as cisco.com, into IP addresses.

Host Config

	DHCPv4 - Dynamic Host Configuration Protocol for IPv4. A DHCPv4 server dynamically assigns IPv4 addressing information to DHCPv4 clients at start-up and allows the addresses to be re-used when no longer needed.
	DHCPv6 - Dynamic Host Configuration Protocol for IPv6. DHCPv6 is similar to DHCPv4. A DHCPv6 server dynamically assigns IPv6 addressing information to DHCPv6 clients at start-up.
	SLAAC - Stateless Address Autoconfiguration. A method that allows a device to obtain its IPv6 addressing information without using a DHCPv6 server.
	
Email

	SMTP - Simple Mail Transfer Protocol. Enables clients to send email to a mail server and enables servers to send email to other servers.
	POP3 - Post Office Protocol version 3. Enables clients to retrieve email from a mail server and download the email to the client's local mail application.
	IMAP - Internet Message Access Protocol. Enables clients to access email stored on a mail server as well as maintaining email on the server.
	
File Transfer

	FTP - File Transfer Protocol. Sets the rules that enable a user on one host to access and transfer files to and from another host over a network. FTP is a reliable, connection-oriented, and acknowledged file delivery protocol.
	SFTP - SSH File Transfer Protocol. As an extension to Secure Shell (SSH) protocol, SFTP can be used to establish a secure file transfer session in which the file transfer is encrypted. SSH is a method for secure remote login that is typically used for accessing the command line of a device.
	TFTP - Trivial File Transfer Protocol. A simple, connectionless file transfer protocol with best-effort, unacknowledged file delivery. It uses less overhead than FTP.
	
Web and Web Service

	HTTP - Hypertext Transfer Protocol. A set of rules for exchanging text, graphic images, sound, video, and other multimedia files on the World Wide Web.
	HTTPS - HTTP Secure. A secure form of HTTP that encrypts the data that is exchanged over the World Wide Web.
	REST - Representational State Transfer. A web service that uses application programming interfaces (APIs) and HTTP requests to create web applications.

Transport layer:

Connection-Oriented

	TCP - Transmission Control Protocol. Enables reliable communication between processes running on separate hosts and provides reliable, acknowledged transmissions that confirm successful delivery.

Connectionless

	UDP - User Datagram Protocol. Enables a process running on one host to send packets to a process running on another host. However, UDP does not confirm successful datagram transmission.

Internet Layer:

Internet Protocol

	IPv4 - Internet Protocol version 4. Receives message segments from the transport layer, packages messages into packets, and addresses packets for end-to-end delivery over a network. IPv4 uses a 32-bit address.
	IPv6 - IP version 6. Similar to IPv4 but uses a 128-bit address.
	NAT - Network Address Translation. Translates IPv4 addresses from a private network into globally unique public IPv4 addresses.

Messaging

	ICMPv4 - Internet Control Message Protocol for IPv4. Provides feedback from a destination host to a source host about errors in packet delivery.
	ICMPv6 - ICMP for IPv6. Similar functionality to ICMPv4 but is used for IPv6 packets.
	ICMPv6 ND - ICMPv6 Neighbor Discovery. Includes four protocol messages that are used for address resolution and duplicate address detection.

Routing Protocols

	OSPF - Open Shortest Path First. Link-state routing protocol that uses a hierarchical design based on areas. OSPF is an open standard interior routing protocol.
	EIGRP - EIGRP - Enhanced Interior Gateway Routing Protocol. An open standard routing protocol developed by Cisco that uses a composite metric based on bandwidth, delay, load and reliability.
	BGP - Border Gateway Protocol. An open standard exterior gateway routing protocol used between Internet Service Providers (ISPs). BGP is also commonly used between ISPs and their large private clients to exchange routing information.

Network Access Layer:

Address Resolution

	ARP - Address Resolution Protocol. Provides dynamic address mapping between an IPv4 address and a hardware address.

	Note: You may see other documentation state that ARP operates at the Internet Layer (OSI Layer 3). However, in this course we state that ARP operates at the Network Access layer (OSI Layer 2) because it's primary purpose is the discover the MAC address of the destination. A MAC address is a Layer 2 address.

Data Link Protocols

	Ethernet - Defines the rules for wiring and signaling standards of the network access layer.
	WLAN - Wireless Local Area Network. Defines the rules for wireless signaling across the 2.4 GHz and 5 GHz radio frequencies.

Layer 3 Logical Address:
	-layeru 3 da 2 adrese IP cea sursa si cea destinatie
	Source IP address : adresa IP a dispozitivului care trimite mesajul
	Destination IP address : adresa IP a dispozitivul care primeste mesajul
	
	-Adresa Ip are doua parti cea care arata Netwroku si cea care arata hostu
	network portion(IPv4) or Prefix(IPv6) : partea din stanga care arata la fel pe LAN e cea care reprezinta networku, e data de subnet mask
	host portion(IPv4) or Interface ID(IPv6) : partea ramasa care identifica un host, este o portoiune unica pe LAN 

Role of the Data Link Layer Addresses:

	-layeru 2 baga adresele MAC la destinatar si sursa
	Source MAC address : unic, o adresa a NIC-ului a hostu-ului care trimite mesajul
	Destination MAC address : unic, o adresa a NIC-ului a hostu-ului care primeste mesajul
	-cand host-urile sunt pe LAN-uri diferite, hostul sursa o sa isi dea seama de asta dupa IP	

Different IP networks:

Data Link Addresses:
	-daca hosturile nu sunt in acelasi LAN, pachetul cu sursa si destinatia IP va fi encapsulat intr-un data-lin frame nou, pentru ca la iesirea din dispozitivele intermediarea se schimba adresa MAC
	Source data link address : adresa NIC a dispozitivului care trimite data link frame-ul
	Destination data link address : adresa NIC a dispozitivului care primeste data link frame-ul, poate fi router switch sau chiar hostu destinatie

Characteristics Copper Cabling:
	-cel mai comun
	-ieftin, usor de montat, resistenta mica la electricitate
	-limitat de distanta si interferente ale semnalului
	-datele sunt transmise sub forma de pulsuri electrice
	-sunt 3 tipuri
	-cu cat semnalu merge mai departe cu atat se deterorieaza mai mult asta inseamna signal attentuation	
	-calitatea semnalului mai poate fi interferata de alte 2 surse:
		-Electromagnetic interference(EMI) or radio frequency interference(RFI):
			-semnale radio si electromagnetice de la alte dispoztive
		-Crosstalk:
			- este cand se amesteca datele de la mai multe circuite apropiate, pentru ca datele transmise creaza un camp magnetic
			- ca sa il eviti unele cabluri de cupru au opposing circuit wire pairs imbinate, care cica nu is afectate de crosstalk

		The susceptibility of copper cables to electronic noise can also be limited using these recommendations:
			-Selecting the cable type or category most suited to a given networking environment
			-Designing a cable infrastructure to avoid known and potential sources of interference in the building structure
			-Using cabling techniques that include the proper handling and termination of the cables

The Data Link Layer:
	-permite layerelor mai mari sa acceseze media
	-acepta data, deobicei de la layeru3, si encapsuleaza in layer2 frames
	-controleaza cum este plasata data si primita pe media
	-schimba frames intre doua host-uri ale internetului
	-primeste data encapsulata, debocei pachete de layer3. si le directioneaza catre protocolul corect
	-error detection, si rejects orice frame corupt
	-prepareaza datele network pentru physical network
	node : un dispozitiv care poate primii, crea, salva si trimite date pe un communications path, un host sau un dispozitiv intermediar
	
IEE 802 LAN/MAN Data Link Sublayers:
	-standardele IEEE 802 LAN/MAN sunt specifice pentru LAN's, WLAN's and WPAN's
	-data link layeru IEEE 802 LAN/MAN contine 2 sublayere:
		-Logical Link Control(LLC):
			-acest sublayer face comunicatia intre network software si upper layers
			-acest sublayer face comunicatia intre network software si dispozitivele hardware
			-encapsuleaza frame-urile	
	-Media Access Control(MAC):
			-da adresele MAC
			-implementeaza acest sublayer in hardware
		
	-sublayerele LLC iau datale ede la layeru3 si adauga informatii de control ale layerului 2
	-sublayeru MAC controleaza NIC-u si alte hardware-uri care sunt responsabile pentru trimiterea si receptionarea datei
	Layeru MAC contine data encapsulation:
		-frame delimiting : dau niste delimitatori, care fac sincronizarea intre transmitting and receiving nodes
		-addressing : dau adresele MAC de sursa si destinatie
		-error detection : baga un trailer la frame care sa vada daca sunt erori

		-Logical topology : se refera la modul de transfer al frame-urilor de la un node la altulm identifica conexiuni virtuale folosinduse de interfatele dispozitivelor si schema adreselor de la layeru 3

Hald and Duplex Communication:

		-doar un dispozitiv poate primii sau transmite data
		
	Full-duplex Communication:
		-ambele dispozitive pot transmite si primii date simultan
		-data link layeru presupune ca media este valabila pentru transmisia datelor pentru ambele node-uri in orice moment
	
Access Control Methods:
	-LAN-urile si WLAN-urile sunt exemple de multiacces networks, adica un network in care doua sau mai multe dispozitive pot accesa internetul simultan
	Access Control Methods:
		-Contention-based access
		-Controlled access
	Contention-based access:
		-toate node-urile opereaza in half-duplex, intrecand-use pentru cine ajunge primu
		exemple:
			-Carrier sense multiple access with collision detection(CSMA/CD) folosit in bus-topology LAN's
			-Carrier sense multiple access with collision avoidance(CSMA/CA) folost in WLAN's
	
	Controlled access:
		-fiecare node are momentul sau pentru a accesa mediumu
		-aceste networkuri sunt ineficiente
		exemple:
			-Legacy Token Ring
			-Legacy ARCNET

Contetion-Based Access - CSMA/CD:
	exmple:
		WLAN, bus-topology LAN, LAN using hub
	-opereaza in modul half-duplex
	-daca doua dispozitive transmit un mesaj deoadata o sa apara o coliziune, care va fi detectata de ambele dispozitive
	-collision detection(CD) part of CSMA/CD
	-NICu compara data transmisa cu data primita
	-data trimisa de dispozitive va fi stricata si va trebuii sa fie retrimisa
	
Contention-Based Access - CSMA/CA:
	-cand un host primeste un frame, o sa zica ca il primeste si un timp 
	-celelalte host-uri o sa stie sa nu faca niciun request in acel timp
	-dupa ce hostu primeste acel frame, o sa zica ca l-o primit
	-celelalte hosturi o sa isi faca requesturile lor

The Frame:

		-header
		-data
		-trailer(doar la layeru 2)
	
Frame Fields:
	Headrul Frameului contine:
		-Frame Start : niste biti care iti zic ca acuma incepe un frame
		-Addressing : adresele MAC sursa si destinatie  
		-Type : ce protocol s-a folosit la Layeru 3
		-Control : un control special al flow-ului datelor
		-Data : pachetu primit de la layeru 3
		-Error Detection : verifica daca frame-ul a fost trimis fara sa fie corupt
		-Frame Stop : niste biti care iti zic ca acuma s-a incheiat frame-ul

Frame Forwarding Methods on Cisco Switches:
	-sunt 2 metode:
	Store-and-forward switching : se vede daca nu sunt erori la trimitere si apoi se trimite
	Cut-through switching : frame-ul este trimis inainte sa ajunga tot, cum s-au citit adresele MAC s-o si trimis
	
Cut-through switching:
	-sunt 2 metode:
	Fast-forward switching : cum a citit adresa MAC destiantie l-a si trimis, deobicei au erori
	Fragment-free switching : se primesc primi 64 de biti, se verifica de erori , si dupa se trimite, pentru ca cele mai multe erori apar in aia 64 de biti

Memory Buffering on Switches:
	-sunt 2 metode:
	Port-based memory: 
		- frameurile sunt tinute intr-un queue specific pentru porturile de primire si trimite
		- un frame este trimis doar dupa ce sunt trimise celelalte din queue
		- exista posibilitatea ca frameurile sa intarzie pentru ca un port e prea aglomerat
	
	Shared Memory:
		- frameurile sunt tinute intr-u buffer comun din care trimit toate porturile , si nu mai trebuie asa memorie buffer mare pentru ca este impartita la cate porturi sunt
		- frameurile in buffer sunt legate dinamic la destinatie, deci un pachet poate fi trimis pe un port si sa vina pe altul

The Network Layer:
	-servicii care le permit host-urilor sa faca schimb de date

		-IPv6
		-acestea trebuie sa faca 4 operatiunii:

			-routing : servicii pentru directionarea pachetelor la o destinatie

Characteristics of IP:
	connectionless : nu se face nicio conexiune cu destinatia pana sa se trimita pachetu
	best effort : face tot posibilu ca pachetu tau sa ajunga, da nu e garantat
	media independent : indepentent de medium

Connectionless: 
	-nu se face nicio conexiune cu hostu destinatie in care sa ii zici ca o sa vina un pachet
	
Best Effort:
	-protocolul IP nu garanteaza ca toate pachetele ajung la destinatie
	-protocolul IP nu afla daca toate pachetele ajung sau daca data a fost stricata in transmisie
	
Media Independent:
	-pachetele pot fi trimise pe orice cablu sau prin orice metoda
	-data link layeru este responsabil pentru prepararea pachetului IP pentru transmisia pe diferite cabluri sau diferite metode
	-daca un pachet e prea mare pentru mediumu in care va fi trimis o sa fie impartit in mai multe pachete de router, acest proces se numeste fragmentare
	-pachetele IPv6 nu pot fi fragmentate

IPv4 Packet Header:
	-este folosit pentru a asigura ca pachetul este trimis la urmatorul sau stop in cursa sa catre destinatie
	
IPv4 Packet Header Fields:
	Version : 4 biti(0100) care identifica ca pachetul e unul IPv4
	Differentiated Services of DiffServ(DS) : 8 biti folositi pentru a determina prioritatea fiecarui pachet
	Time to Live(TTL) : 8 biti folositi pentru a limita viata unui pachet, adica sursa de un nr, la fiecare router acel nr scade cu unu, daca la vre-un router valoarea e 0 acesta dropeaza pachetu si il cere din nou dar cu TTL mai mare
	Protocol  : 8 biti folositi pentru a identifica protocolul folosit in layerul anterior(4)
	Header Checksum : folosit sa vada daca au aparut erori in headeru IPv4
	Source IP address : 32 biti care exprima sursa IPv4 a sursei 
	Destination IP address : 32 biti care exprima sursa IPv4 a destinatiei 

IPv6 Overview:
	increased address space : IPv6 128 biti > IPv4 32 biti
	improvved packet handling : headru IPv6 are mai putine fielduri de completat
	eliminates the need for NAT : nu mai trebuie NAT ca o sa fie destule adrese IP

IPv6 Packet Header:
	-contine:
		-version
		-traffic class
		-payload length
		-next header
		-hop limit
		-source ip address
		-destination ip address
	-are 40 de bytes

Fields in the IPv6 Packet Header:
	version : 4 biti(0110) care identifica ca pachetu asta e unu de IPv6
	traffic class : 8biti echivalentul a IPv4 DS field
	flow label : 20biti sugereaza ca toate pachetele cu acelasi flow label au aceasi prioriatate la routere
	payload length : 16biti indica dimensiunea datei
	next header : 8biti echivalentul a IPv4 Protocol Field
	hop limit : 8biti echivalentul a IPv4 TTL
	source IPv6 address : addresa IPv6 a sursei
	destination IPv6 address : addresa IPv6 a destinatiei

ARP overview:
	-ii dai un IP si iti da adresa MAC a acelui IP
	are doua functii:
		-resolving IPv4 addresses to MAC addresses
		-maintaining a table of IPv4 to MAC address mappings

ARP functions:
	-cand un pachet e trimis la data link layer ca sa fie encapsulat, acesta se uita in tabelul ARP ca sa gaseasca adresa MAC corespunzatoare acelui IP
	-daca nu este regasit in tabel o sa se trimita un ARP request
	-daca este pe LAN o sa caute adresa acelui IP
	-daca este WAN o sa caute adresa MAC a default gatewayu a routerului

ARP request:
	-daca nu se regaseste adresa MAC in tabelu ARP o sa se trimita un ARP request
	arp request : un broadcast, care este un frame dar fara IPv4 header, ca header information are sursa si destinatia MAC si type-ul mesajului adica ARP message care are type field = 0x806, type fieldu ii zice la NIC ca aceasta data trebuie sa fie trecuta ca ARP process
	frameul pe care il trimite consta in:
		-ethernet header:
			-destination MAC : FF-FF(adresa broadcast)
			-source MAC : adresa MAC a hostului
		-ARP request:
			-target IPv4 : adresa IPv4 a hostului la care vrem sa-i facem rost de adresa MAC
			-target MAC : ???(nu este nicio valoare)
	-hostu care are target IPv4-le egal cu IPv4-le sau o sa tirmita un ARP reply

ARP reply:
	-headeru contine : destination MAC address, source MAC address, si type ARP message (0x086)
	-daca nu este ARP reply pachetul o sa fie sters
	frame-ul in care este ARP replyu contine:
		-TINE CONT CA ASTEA IS INFO DE LA HOSTU CARE DA REPLYU
		-Ethernet header:
			-destination MAC address
			-source MAC address
		-ARP reply:
			-sender IPv4
			-sender MAC

	-si dupa se saleaza sender MACu in tabelu ARP cu sender IPv4 si apoi se pot face frame-uri cu acele adrese
	-ARPu poate fi si intre host si router

Removing from ARP table:
	-o pereche de IP si MAC o sa stea in tabel undeva de la 15 la 45 de secunde

			-target MAC(MACu de la hostu care recunoaste adresa IPv6)
	-hostu care o facut requestu primeste acest frame si salveaza ce este in ICMPv6 intr-un tabel DN(ca un tabel ARP numa ca pt IPv6)

The Subnet Mask:

The Prefix length:
	-adresa IPv4/x
	- x-ul arata cati biti al subnet mask-ului au valoarea 1

Determining the Network - Logical AND:
	1 AND 1 = 1
	-ca sa afli network addressu iei primu element de la ipv4 host address si subnet masku, si faci primu caracter cu primu si daca ambele is 1 bagi 1 si daca nu bagi 0, si daca iei tot asa la final o sa ai adresa network IPv4 in biti

IPv6 Addressing Formats:
	-are 128 de biti
	-e scris sub forma de 32 cifre hexadeciamale, in octete
	-poti sa nu scrii 0-urile din fata ca sa fie mai scurta
	-poti octetele care au doar 0-uri succesive sa le scrii ::
	-mai multe octete de 0 succesive vor fi scrise ca ::
	-si pula mea ma celelalte care is mai putine ii succesivitate le scrii : 0:
	
Unicast, Multicast, Anycast:
	unicast - doar la un host
	multicast - un pachet la mai multe dispozitive care fac parte dintr-un grup
	anycast - is mai multe host-uri cu acelasi unicast address si tu trimiti pachetu si care e cel mai aproape de tine il primeste

		Link-local Address(LLA) : trebuie sa aiba orice IPv6-enabled device, folosite la convorbirea prin LAN 

IPv6 GUA:
	-primul xextet incepce cu 001
	-de la 0010 0000 0000 0000(2000) la 0011 1111 1111 1111(3fff)
 	
IPv6 Address with a /48 Global Routing Prefix and /64 Prefix:
	-adresa IPv6 GUA contine:
		-global routing prefix(48 biti)
		-subnet id(16 biti)
		-interface id(64 biti)(portiunea care arata adresa hostului)

IPv6 GUA Structure:
	-global routing prefix : portiunea network(prefix) care este setata de ISP, la un utilizator sau site, deobicei la utilizatori este prefixul este /48
	-subnet id : folosit de organizatii ca sa identifice subnet-uri pe site
	-interface id : partea host a adresei, este interface pentru ca poate un host are mai multe mufe de conectare

IPv6 LLA:
	-ii neaparata, ca gen cu asta comunici pe subnetu in care esti
	-de la fe80::10	la febf(::10, poate nu is sigur ne zice doar de primu hextet)
	-un dispozitiv poate primii adresa LLA static sau dynamic(si-o creaza el automat)
	
	
Static GUA Configuration on a Windows Host:
	-adresele GUA pot fi ori static ori automatic(SLAAC sau DHCPv6)

RS and RA messages:
	-cand un host vrea sa faca rost de o adresa IPv6 dynamic, o sa aiba loc un schimb de mesaje intre router si host
	RA : Router Advertisment
	RS : Router Solicitant
	- RS-u il trimite hostu la router ca sa ii ceara adrese
	- RA-u il trimite routeru la host care contine network prefix, prefix-length si default-gateway information
	-un mesaj RA contine:
		-network prefix si prefix length : ii zice la dispozitiv in ce network 
		-default gateway address : IPv6 LLA, adresa sursa IPv6 a mesajului RA
		-DNS addresses and domain name : adresele serverelor DNS si un nume de domeniu
	-sunt 3 metode pentru mesaje RA:
		-SLAAC : "Am tot ce ai nevoie prefixul, lungimea prefixului si default gateway address"
		-SLAAC with a stateless DHCPv6 server : "Aicia is informatile mele, dar trebuie sa mai iei informatii de la un DHCPv6 server, cum ar fi adrese DNS"
		-Stateful DHCPv6(no SLAAC) : "Iti pot da default gateway addressu, dar trebuie sa intrebi un stateful DHCPv6 server celelalte informatii	
	
SLAAC:
	-permite dispozitivelor sa isi creeze proprile informatii GUA fara servicile DHCPv6, insa sunt dependente de mesajele RA ICMPv6 primite de la router
	adresa GUA:
		prefixu din mesaju RA
		EUI-64 sau isi genereaza un numar random de 64 biti

SLAAC and Stateless DHCPv6:
	-foloseste SLAAC sa isi creeze adresa GUA
	-adresa LLA a routerului ca si default gateway
	-si foloseste un stateless server DHCPv6 ca sa obtina informatii despre serveru DNS si un domain name
	stateless DHCPv6 server : distribuie adrese de la servere DNS si domain names

Stateful DHCPv6:
	-adresa LLA a routerului ca si default gateway
	-si DNS server, domain name si alte info le da un server stateful DHCPv6
	
EUI-64 Process vs Randomly Generated:
	-cand mesajul RA este SLAAC sau SLAAC cu stateless DHCPv6, clientul trebuie sa isi genereze singur interface id-u
	-interface id-u poate fi creat ori prin EUI-64 sau isi genereaza un numar random de 64 biti

EUI-64 Process:
	-interface id prin EUI-64 este reprezentata prin binary
	-ia adresa MAC si adauga in mijlocu ei alti 16 biti
	adresele MAC deobicei sunt reprezentate hexadeciaml si sunt facute din doua parti:
		-Organizationally Unique Identifier (OUI) : un cod de 24 biti, 6 cifre hexadecimale, primit de la IEEE
		-Device identifier : o valoare unica de 24 biti, 6 cifre hexadecimale intr-un OUI comun.
	-interface id facut prin EUI-64 are 3 parti:
		-24 biti OUI de la adresa MAc, dar al 7-lea bit este invartit(daca e 0 se face 1 si vice versa)
		-aia 16 biti random
		-24 biti Device Identifier de la adresa MAC

Randomly Generated Interface IDs:
	-se generaza un nr de 64 biti random
	-windows foloseste acest proces
	-si apoi adresa se combina cu prefixu si se face GUA-ul

Dynamic LLA's:
	-este creat folosind fe80::/10 prefix si interface id
	
Assigned IPv6 Multicast Addresses:
	-adresele IPv6 multicast au prefixu ff00::/8
	sunt doua tipuri:
		- well-known
		-solicited node

Well-known Multicast Addresses:
	sunt 2 grupe:
		ff02::1 All-nodes multicast group : e ca un broadcast la IPv4, e un mesaj precesat de toate dispozitivele
		ff02::2 All-routers multicast group : un grup multicast pentru routere, doar routerele IPv6

Solicited-Node IPv6 Multicast Addresses:
	-similar la All-nodes multicast group
	-avantajul insa este ca este configurat la o adresa Ethernet multicast, ca un multicast la IPv4
	-16-bit subnet ID - Creates up to 65,536 subnets.
	-64-bit interface ID - Supports up to 18 quintillion host IPv6 addresses per subnet (i.e., 18,000,000,000,000,000,000).
